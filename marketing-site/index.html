The problem

Mainstream tools assume bandwidth you don't have.

They recommend 600 kbps–1.8 Mbps for 1:1 video, treat your screen as a blurry video stream, tie the remote cursor to late frames, and fall apart under the bursty 1–5% packet loss of real radio links. The result: support by phone call plus screenshots over email — in 2026.

What you get

One session. Everything an assist needs.

🎙

Voice that survives loss

Opus 1.5 with DRED deep redundancy reconstructs speech through bursty loss. At 5% loss and 300 ms RTT: zero audible gaps. Silence costs ≈ 0 kbps with DTX.

🖥

Screens that stay legible

Text is sent as text deltas, not video. Terminals, logs, and error dialogs render sharper than video-codec competitors while using 3–10× less bandwidth.

🖱

A cursor that obeys

Remote control rides its own reliable channel, decoupled from video frames. Input feels immediate even at 300 ms RTT — no more overshooting buttons.

📦

File push that resumes

FastCDC chunking, BLAKE3 verification, and RaptorQ fountain codes. Transfers survive drops and roaming, resume where they left off, and never block voice.

⚙️

A governor, not a guess

A 10 Hz control loop watches bandwidth, RTT, loss, and thermal state, and shifts codec gears per stream. Downgrades in < 200 ms; upgrades only on validated headroom.

🔐

E2EE with no opt-out

Media never touches a server. TURN relays, used only as a last resort, forward ciphertext they cannot read. Neural-reconstructed media is always labeled.

Adaptive quality

Four tiers. The session never just dies.

Every 100 ms the governor re-budgets each stream with a strict priority order — audio first (6 kbps floor), control second, screen third, bulk transfer last. As your link collapses, LowBand sheds load instead of dropping the call.

FULL ≥ 300 kbps

Camera video, high-fps screen, generous bulk transfer.

COMFORTABLE 128–300 kbps

Pleasant everything — smooth screen, wideband voice, steady transfers.

CONSTRAINED 48–128 kbps

The design point: clear voice, legible screen deltas, responsive control.

SURVIVAL < 48 kbps

Voice floor + coarse text deltas + control. Refinements pause; the session holds.

Not just support — calls

Talk face to face across the world — even on 3G.

The same engine that survives a support session is a full-duplex voice and video call, one-to-one or a group of up to four. There's no cloud media server in the middle: your audio and video go straight peer-to-peer over the shortest network path. That direct hop — not a round trip through a data center on another continent — is what keeps latency low when both of you are on mobile data or ADSL2, half a planet apart.

01

Point both sides at a rendezvous

A tiny signaling server — yours, or a shared community one — introduces the two ends and then steps out of the path. It only swaps connection details; it never sees a byte of your call. Add a STUN server and LowBand punches through home routers and carrier NAT, so two phones on opposite sides of the world connect directly, no relay.

both machines
# build the daemon with real mic/speaker support
$ cargo build -p lowbandd --features audio

# a rendezvous both ends can reach (HOST:PORT, not a URL)
# + a public STUN so two NATed networks connect directly
SIG=relay.example:3478   STUN=stun.l.google.com:19302
02

One person hosts and shares a code

Whoever starts the call gets a short code to pass along — text it, say it, put it in a calendar invite. The handshake is Noise-IK end-to-end encrypted and bound to that code, so only the person you gave it to can join.

Amara · Lagos
$ lowbandd --signaling $SIG --stun $STUN --host
lowbandd: hosting session — join code: R4M-8KP
lowbandd: waiting for peer…   (share the code any way you like)
03

The others join — direct, encrypted, live

Enter the code and you're connected peer-to-peer. Voice is Opus with DRED, so a one-second burst of packet loss on a moving train stays inaudible; on a truly bad link the governor sheds video and camera first and holds the voice floor. The call degrades — it doesn't drop.

Diego · Buenos Aires
$ lowbandd --signaling $SIG --stun $STUN --join R4M-8KP
lowbandd: secure channel established (direct p2p, 0 relays)

# voice: opus+dred 16 kbps · no gaps
# link:  72 kbps · rtt 240 ms · tier CONSTRAINED
04

Make it a group — up to four

Host a room instead and everyone joins the same code. LowBand builds a full mesh — every person connects directly to every other, so no one's audio detours through a central server. The uplink budget is split evenly across the call, and incoming voices are mixed locally. Four people, four corners of the world, one conversation.

group of four
# host opens a room for up to 4
$ lowbandd --signaling $SIG --stun $STUN \
    --room --room-id amara --room-size 4
lowbandd: hosting mesh room (size 4) — room code: 92C-5TD

# everyone else joins with their own id
$ lowbandd --signaling $SIG --stun $STUN \
    --room-join 92C-5TD --room-id diego --room-size 4
lowbandd: mesh established as 'diego' — 3 peer(s), 133333 bps/peer

Prefer a single command? The voice-call.sh helper wraps all of this — voice-call.sh host, join <code>, or mesh — with SIGNALING_URL, STUN, and AUDIO=1 as environment knobs.

🛰

The short path wins

Media never rides through a cloud region. The packets take the most direct route the two networks share — the single biggest lever on real-world latency between distant callers.

🎚

Audio always goes first

The 10 Hz governor budgets voice ahead of everything else with a 6 kbps floor, so conversation stays snappy and intelligible even as a 3G or ADSL2 link wobbles.

🧩

Loss doesn't mean gaps

Opus DRED reconstructs speech through bursts of up to a second of loss, and DTX drops silence to ≈ 0 kbps — so a shaky uplink sounds smooth and costs almost nothing when no one's talking.

Tutorial

From zero to fixed in four steps.

01

Install the daemon

One binary per platform. IT teams deploy silently via MSI or pkg; everyone else grabs the installer or builds from source. No account required to receive help.

terminal
# macOS / Linux
curl -fsSL https://get.lowband.dev | sh

# Windows (silent deploy)
msiexec /i lowband.msi /qn

# or from source (with mic/speaker support)
cargo build -p lowbandd --features audio
02

Start a session, share a short code

The technician starts an assist session and gets a short join code to read over the phone. Signaling is a stateless rendezvous — it never sees your media. NAT traversal is automatic; over 85% of sessions connect directly P2P.

technician
$ lowbandd --signaling relay.example:3478 --host
lowbandd: hosting session — join code: K7F-2QX
lowbandd: waiting for peer…
03

The assisted side consents — explicitly

One clear consent screen. View, control, file transfer, and clipboard are separate, individually revocable grants — never a blanket "allow everything". A persistent indicator shows help is active, and a panic key cuts control on both sides in under 50 ms.

assisted user
$ lowbandd --signaling relay.example:3478 --join K7F-2QX

  Tan (Acme IT) is asking to:
  [✓] view your screen
  [✓] control mouse & keyboard
  [ ] send & receive files
  [ ] sync clipboard

  Enter accept selected  ·  Esc decline
  panic key: press Ctrl+Shift+End anytime
04

Work the session — then prove it happened

A live, honest quality indicator shows the current tier, bitrate, RTT, and loss. Push files that resume after drops. When you're done, export a signed, tamper-evident audit log of who did what, when — compliance-ready JSON.

in session
$ lowband send ./fix-driver.zip
✓ 8.4 MB · resumable · verified blake3

$ lowband audit export --session K7F-2QX
✓ audit-K7F-2QX.json (ed25519 signed)
  grants, input events, transfers, timestamps

Trust, engineered

Deployable by teams that read the spec sheet.

E2EE, always

End-to-end encryption with no opt-out. No media server exists to compromise; relays forward ciphertext only.

Scoped, revocable consent

View / control / file / clipboard are separate capability grants, enforced per event — revoke any of them mid-session.

Signed audit logs

Every session can export a tamper-evident, signed record: identity keys, grants, timestamps, actions.

Labeled AI media

When survival tier uses neural voice or reconstructed head video, it is clearly labeled. Never silently synthetic.
