<project_specification mode="greenfield">
  <project_name>lowband</project_name>

  <overview>
    LowBand is a peer-to-peer remote-support and conferencing product engineered for the
    networks incumbents gave up on — 3G, ADSL2, congested rural broadband, and metered mobile.
    It carries voice, screen view, remote control, and file transfer in one E2EE session that
    stays fully functional at 64 kbps and pleasant at 150 kbps per direction, roughly a quarter
    of the bandwidth floor mainstream conferencing tools publish for 1:1 video. The wedge is
    remote IT assistance, where the person who most needs help is disproportionately the person
    on the worst connection.
  </overview>

  <target_audience>
    Tan, the MSP/internal-IT technician running 8–15 sessions a day to poorly connected
    endpoints; Ana, the low-confidence assisted user on a weak, metered connection who needs a
    short join code, one clear consent screen, and a panic key; and Mo, the IT manager who buys
    on E2EE with no media touching vendor servers, scoped per-capability consent, signed audit
    logs, and silent MSI/pkg deployment.
  </target_audience>

  <technology_stack>
    <frontend>
      <framework>Native per-platform UI shells (Windows, macOS, Linux) over local IPC</framework>
      <language>Platform-native (Win32/WinUI, SwiftUI/AppKit, GTK) rendering + consent dialogs</language>
      <styling>Trust-first minimal UI: persistent consent indicator, honest quality bar</styling>
      <state_management>IPC event stream from lowbandd (capnp/flatbuffer schema)</state_management>
      <routing>Screens: join-by-code, consent, in-session, settings, audit export</routing>
    </frontend>
    <backend>
      <framework>Rust core daemon (lowbandd): lbtp, crypto, gov, audio, video, screen, control, xfer, nn, obs</framework>
      <codecs>libopus 1.5+ (DRED), SVT-AV1 + dav1d, neural vocoder + talking-head via ONNX Runtime</codecs>
      <transport>LBTP over UDP — custom protocol borrowing from QUIC and RTP, ICE-based NAT traversal</transport>
      <compression>zstd + pre-trained dictionaries, BLAKE3, FastCDC, RaptorQ fountain codes</compression>
    </backend>
    <database>
      <primary>Embedded SQLite local state store in lowbandd (no server database, no media server)</primary>
      <dev>Same embedded store; tables: known_peers, session_records, audit_log, chunk_cache</dev>
    </database>
    <communication>
      <api_style>P2P media/control over LBTP UDP; stateless WebSocket signaling for rendezvous</api_style>
      <response_format>Binary LBTP frames on channels 0–8; signaling exchanges SDP-like blobs and ICE candidates</response_format>
    </communication>
    <infrastructure>
      <signaling>Stateless WebSocket rendezvous service (session codes, offer/answer, ICE relay)</signaling>
      <relay>coturn TURN fleet, last resort only, forwards E2EE ciphertext it cannot read</relay>
      <neural_runtime>ONNX Runtime with CoreML / NNAPI / DirectML / CPU execution providers</neural_runtime>
    </infrastructure>
  </technology_stack>

  <prerequisites>
    - Rust stable toolchain (cargo) for the lowbandd core daemon
    - C toolchain + FFI libs: libopus >= 1.5, SVT-AV1, dav1d, zstd, BLAKE3, a RaptorQ library
    - ONNX Runtime with platform execution providers (CoreML/NNAPI/DirectML/CPU)
    - Platform capture/injection SDKs: DXGI + SendInput (Win), ScreenCaptureKit + CGEvent (macOS), PipeWire + libei/XTest (Linux)
    - A coturn TURN server and a stateless WebSocket signaling deployment for integration tests
    - Environment variables: SIGNALING_URL, TURN_URL, TURN_SHARED_SECRET (document in .env.example)
    - netem / mahimahi for the network-emulation verification harness
  </prerequisites>

  <database_schema>
    <tables>

      <table name="known_peers">
        <column>id UUID PRIMARY KEY DEFAULT gen_random_uuid()</column>
        <column>static_pubkey BLOB NOT NULL UNIQUE</column>
        <column>display_label VARCHAR(120)</column>
        <column>first_seen_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()</column>
        <column>last_seen_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()</column>
      </table>

      <table name="session_records">
        <column>id UUID PRIMARY KEY DEFAULT gen_random_uuid()</column>
        <column>connection_id BIGINT NOT NULL</column>
        <column>peer_id UUID REFERENCES known_peers(id) ON DELETE SET NULL</column>
        <column>started_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()</column>
        <column>ended_at TIMESTAMP WITH TIME ZONE</column>
        <column>peak_tier VARCHAR(16)</column>
        <column>total_bytes BIGINT DEFAULT 0</column>
      </table>

      <table name="audit_log">
        <column>id UUID PRIMARY KEY DEFAULT gen_random_uuid()</column>
        <column>session_id UUID NOT NULL REFERENCES session_records(id) ON DELETE CASCADE</column>
        <column>event_type VARCHAR(48) NOT NULL</column>
        <column>capability VARCHAR(16)</column>
        <column>occurred_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()</column>
        <column>signature BLOB NOT NULL</column>
      </table>

      <table name="chunk_cache">
        <column>blake3_hash BLOB PRIMARY KEY</column>
        <column>peer_id UUID REFERENCES known_peers(id) ON DELETE CASCADE</column>
        <column>byte_len INTEGER NOT NULL</column>
        <column>zstd_payload BLOB NOT NULL</column>
        <column>last_used_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()</column>
      </table>

      <table name="session_codes">
        <column>code CHAR(9) PRIMARY KEY</column>
        <column>responder_pubkey BLOB NOT NULL</column>
        <column>created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()</column>
        <column>expires_at TIMESTAMP WITH TIME ZONE NOT NULL</column>
      </table>

    </tables>
  </database_schema>

  <ui_layout>
    <structure>
      Two native shells per platform (technician and assisted) talk to lowbandd over a local
      IPC socket (flatbuffer/capnp schema). The daemon owns network, crypto, codecs, capture,
      and injection; the shell owns rendering, consent dialogs, and settings, so a shell crash
      never drops the call.

      Screens:
        join            - 9-digit code / link entry; zero networking questions
        consent         - per-capability grant toggles (view / control / file / clipboard)
        in-session      - screen view, remote cursor, voice, honest quality bar, panic key,
                          persistent "you are being helped" indicator, AI-reconstructed badge
        settings        - identity keys, device capability (NPU/gears), data-usage preferences
        audit-export    - review and export the signed session audit_log as JSON
    </structure>
  </ui_layout>
