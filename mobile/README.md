# LowBand mobile viewers (pre-flight)

Assisted-side mobile clients for LowBand — the "Ana joins from her phone"
half of a session. v0.1 is a **pre-flight preview**: join-code entry and the
consent-first session screen. Live LBTP transport arrives when `core/lbtp`
is linked over FFI (see the PRD's v1.1 milestone).

CI: `.github/workflows/mobile-preflight.yml` builds both apps on every push
touching `mobile/**`, and uploads to the stores' pre-release tracks when the
signing secrets exist (steps skip cleanly otherwise).

## Android (`mobile/android`)

Plain Gradle + Kotlin app, package `dev.lowband.viewer`, minSdk 26.

```sh
cd mobile/android
gradle assembleDebug          # debug APK, auto-signed with the debug key
gradle bundleRelease          # release AAB (unsigned without a keystore)
```

Pre-flight release = **Play internal testing track** (up to 100 testers,
no review):
1. Create the app in Play Console (one-time, $25 developer fee).
2. Create an upload keystore: `keytool -genkeypair -v -keystore upload.keystore -alias lowband -keyalg RSA -keysize 4096 -validity 10000`
3. Add the CI secrets listed at the top of the workflow, including a Play
   service account JSON with "Release to testing tracks" permission.
4. Push to `main` — CI signs, uploads to the internal track, and testers
   get it via the opt-in link within minutes.

## iOS (`mobile/ios`)

SwiftUI app, project generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen)
from `project.yml` (the `.xcodeproj` is never committed):

```sh
cd mobile/ios
xcodegen generate
open LowBandViewer.xcodeproj    # macOS only
```

Pre-flight release = **TestFlight**:
1. Join the Apple Developer Program ($99/yr); create the app record in
   App Store Connect with bundle id `dev.lowband.viewer`.
2. Create an App Store Connect **API key** (Keys → App Manager role) and add
   the four `APP_STORE_CONNECT_*` / `XCODE_DEVELOPMENT_TEAM` CI secrets.
3. Push to `main` — CI archives with cloud signing
   (`-allowProvisioningUpdates`) and uploads; internal TestFlight testers
   (your team, up to 100) need no review, external testers need one short
   beta review.

Without secrets, CI still compile-validates the app against the iOS
simulator SDK, so breakage is caught on every push.

## The licensing gate (before any App Store release)

The repo is AGPL-3.0. TestFlight *internal* distribution is fine, but App
Store (and external-TestFlight) distribution of AGPL code conflicts with
Apple's terms — the iOS app must be dual-licensed first, which requires a
CLA from all contributors. Google Play accepts AGPL. Details:
`docs/go-to-market.md` §4.
